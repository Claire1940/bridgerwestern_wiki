## Metode Scan per Lokasi

Kebanyakan spawn yang terlewat terjadi karena pemain hanya lewat di satu titik alih-alih menscan dengan benar. Gunakan micro-routine berulang ini di setiap spawn:

### 1) Bukit Kaktus Greenland
- Dekati dari sudut kecil agar siluet puncak bukit terlihat.
- Cek cover sisi kaktus terlebih dulu.
- Putar 180° sekali sebelum pergi.

### 2) Titik Pinggiran Greenland
- Ikuti tikungan jalur dan berhenti sebentar di perubahan elevasi.
- Gunakan sapuan kamera orang ketiga untuk mengecek garis semak.
- Jika ramai pemain, kurangi waktu berhenti dan rotasi.

### 3) Outlaw Hills
- Mulai dari atas, lalu scan ke bawah.
- Waspadai clipping spawn di dekat punggungan medan.
- Cek ulang kenaikan tengah sebelum pergi.

### 4) Pohon Kabin Ringo
- Prioritaskan pohon di samping kabin sebagai anchor.
- Intip sisi menghadap batang dan sisi menghadap kabin.
- Jika mob hostile aktif, bersihkan cepat lalu cek ulang.

### 5) Interior Gereja
- Masuk langsung dan bersihkan jalur tengah.
- Cek dekat baris bangku/dinding tergantung layout.
- Keluar lalu masuk sekali lagi jika area masih loading saat kamu tiba.

### 6) Checkpoint Barat Terakhir
- Lakukan satu putaran perimeter dan satu lintasan garis tengah.
- Jangan terlalu lama commit jika tidak ada tanda dengan cepat.
- Gunakan titik ini sebagai node keputusan server-hop.

> **Peringatan:** Jika tim lain camping permanen di satu atau dua titik Gyro, data siklus spawn kamu jadi bias. Rotasi server lebih cepat alih-alih memaksakan map yang sama dan diperebutkan.

## Timing Respawn, Server Hopping, dan Hitungan Efisiensi

Bahkan saat kamu tahu semua **gyro spawn locations**, hasil bisa terasa tidak konsisten jika timing kamu meleset. Perlakukan ini seperti loop farming dengan KPI yang bisa diukur:

- **Checks per hour**
- **Encounters per hour**
- **Rata-rata durasi loop**
- **Waktu menunggu vs bergerak**

Baseline praktis untuk 2026:
- Targetkan **10–16 cek lokasi penuh per jam** (tergantung mount/alat speed dan populasi server).
- Jika kamu menyelesaikan 2 loop penuh tanpa encounter, evaluasi kondisi hop.

| KPI | Target Pemula | Target Teroptimasi | Perbaiki Jika Di Bawah Target |
|---|---|---|---|
| **Waktu Loop** | 14–18 menit | 9–13 menit | Kurangi waktu idle di stop bernilai rendah |
| **Checks/Jam** | 8–10 | 10–16 | Pakai timer scan ketat per titik |
| **Encounters/Jam** | 1–2 | 2–4+ | Tingkatkan disiplin rute + aturan hop |
| **Menunggu Idle** | &lt;30% | &lt;20% | Bergerak alih-alih camping satu titik |

### Set Aturan Server-Hop (Sederhana dan Efektif)

Gunakan pohon keputusan ini:
1. **Tidak ada Gyro setelah dua loop penuh?** Hop.
2. **Titik kunci sangat diperebutkan?** Hop.
3. **Lag perjalanan atau pop-in terlambat?** Hop.
4. **Waktu loop bagus + kontestasi rendah?** Tetap satu siklus lagi.

Ini menjaga farming kamu tetap objective-first. Hop acak setiap 5 menit biasanya lebih merusak konsistensi daripada membantu.
